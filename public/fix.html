🚨 Site Down — Emergency Fix Guide

PHP is timing out (504). Only HTML files load. Fix this entirely through cPanel File Manager — no PHP needed.

Step 1 — Delete stale cache files DO THIS FIRST

These files override your .env and are causing the hang. Delete all of them:

In cPanel → File Manager, go to your site root, then into bootstrap/cache/ and delete:

  • config.php  ← most likely culprit
  • packages.php
  • services.php
  • routes-v7.php
  • events.php

Do NOT delete .gitignore — leave that one.

Also check bootstrap/cache/filament/ and delete everything inside it too.

Step 2 — Verify your .env file CRITICAL

In cPanel → File Manager, go to your site root and confirm:

  • File is named exactly .env (dot-env, hidden file, no extension)
  • It is in the root folder — same level as artisan and composer.json
  • NOT inside public/

Click Settings (top right) in File Manager → check "Show Hidden Files" to see it.

Open the .env file and make sure these lines say exactly:

  • BROADCAST_CONNECTION=log
  • REDIS_CLIENT=predis
  • CACHE_STORE=file
  • APP_URL=https://heartsconnect.cc

Step 3 — Delete framework cache too

In cPanel → File Manager, also delete all files inside:

  • storage/framework/cache/data/ — delete all files inside
  • storage/framework/views/ — delete all .php files inside

Step 4 — Test

After deleting the cache files and verifying .env, visit:

  • https://heartsconnect.cc/test.php — should show PHP version instantly
  • https://heartsconnect.cc — main site
If test.php still gives 504, the issue is your cPanel PHP version. Go to cPanel → MultiPHP Manager and make sure PHP 8.2 is selected for your domain.

Step 5 — Check PHP version

Go to cPanel → MultiPHP Manager:

  • Find heartsconnect.cc in the list
  • Make sure PHP version is 8.2 or 8.3
  • If it shows PHP 7.x or 5.x — change it to 8.2 and click Apply

Delete this file once site is working.
