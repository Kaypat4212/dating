🔧 Fix Laravel Echo & Reverb

❌ Current Problem:
Your console shows: 🔴 Laravel Echo initialized with Pusher
This means your frontend assets are still using the old Pusher configuration.

✅ Solution (Choose One):

Option 1: Run Fix Script (Automated)

SSH to your server and run:

cd /home/heartsco/public_html
bash fix-reverb-echo.sh

⚠️ This will take 30-90 seconds (rebuilding assets)

Option 2: Manual Fix (If Script Fails)

Step 1: Edit .env file, make sure these lines exist:

BROADCAST_CONNECTION=reverb
VITE_BROADCAST_DRIVER=reverb
VITE_REVERB_SCHEME=https
VITE_REVERB_HOST=heartsconnect.cc
VITE_REVERB_PORT=443
REVERB_SCHEME=https
REVERB_HOST=heartsconnect.cc
REVERB_PORT=443

Step 2: Clear caches:

php artisan config:clear
php artisan cache:clear
php artisan view:clear
php artisan route:clear

Step 3: CRITICAL - Rebuild frontend assets:

npm run build

⚠️ This step is MANDATORY. Without it, Echo will still use Pusher!

Step 4: Restart Reverb:

bash stop-reverb.sh
bash start-reverb.sh

Option 3: Use Public Cache Clearing Tool

If you don't have SSH access:

  1. Visit: https://heartsconnect.cc/clear-all-cache.php
  2. This clears Laravel caches
  3. But you still need to run npm run build somehow

🧪 Verify It Worked:

After running the fix, refresh your browser (Ctrl+Shift+R) and check console:

✅ Should See:

🟢 Laravel Echo initialized with Reverb

❌ Should NOT See:

🔴 Laravel Echo initialized with Pusher
WebSocket connection to 'wss://ws-us3.pusher.com/...

📱 What Will Work After Fix:

  • ✅ Real-time notifications (instant)
  • ✅ Live message updates (no refresh needed)
  • ✅ Snap notifications on chat page
  • ✅ Online status updates (real-time)
  • ✅ Typing indicators
  • ✅ Voice/Video calls signaling
  • ✅ PWA badge updates

⚠️ Important Notes:

  • npm run build is CRITICAL - Without rebuilding frontend assets, Echo will keep using Pusher config
  • Takes time - Asset rebuild takes 30-90 seconds, be patient
  • Hard refresh needed - After fix, do Ctrl+Shift+R to clear browser cache
  • Reverb must be running - Check with: ps aux | grep reverb

🆘 Still Not Working?

Check these diagnostics:

  1. Is Reverb running? ps aux | grep reverb
  2. Check Reverb logs: tail -50 storage/logs/reverb.log
  3. Verify .env: grep BROADCAST .env && grep REVERB .env
  4. Check if assets were rebuilt: ls -lh public/build/manifest.json
  5. Test WebSocket: Run Diagnostic Tool
